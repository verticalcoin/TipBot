## 2024-05-21 - [Sensitive Information Leakage via Error Messages]
**Vulnerability:** Internal RPC errors were directly passed to Discord users via `message.reply(err.message)`. These messages could leak sensitive system context, file paths, credentials, or daemon specifics. In addition, the lack of `.catch()` handlers could cause unhandled promise rejections and crash the bot entirely.
**Learning:** Returning unhandled or verbose internal error details in external APIs/responses often leaks sensitive internal information which can aid attackers in reconnaissance. This codebase lacked defensive programming around generic message sends, leading to both a vulnerability in data leakage and an availability vulnerability due to process crashing from unhandled promises.
**Prevention:** Never surface raw `.message` attributes directly to end-users unless properly scrubbed. Use generic error messages like "An error occurred." Provide proper global error boundary catching or robust local `.catch()` handlers when making asynchronous external calls (e.g. Discord API calls).
